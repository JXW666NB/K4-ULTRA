## 快速入门

1. 打开 K4 Ultra 编辑器
2. 点击右上角的 **设置按钮**（齿轮图标）
3. 选择 **个性化** 菜单
4. 在各个分类中选择你喜欢的预设样式
5. 所有修改会自动保存，重启后依然生效

## 配置文件位置

K4 Ultra 的个性化配置存储在浏览器的 `localStorage` 中。在不同系统下，配置文件位置如下：

### Windows
```
%APPDATA%\k4-ultra\Local Storage\leveldb\
```
或者
```
%APPDATA%\kitten4\Local Storage\leveldb\
```

### macOS
```
~/Library/Application Support/k4-ultra/Local Storage/leveldb/
```

### Linux
```
~/.config/k4-ultra/Local Storage/leveldb/
```

### 配置项说明

配置以 JSON 格式存储在 localStorage 中，包含以下键名：

| 键名 | 说明 |
|------|------|
| `k4ultra_progress_config` | 进度条配置 |
| `k4ultra_progress_skin` | 进度条自定义CSS皮肤 |
| `k4ultra_skin_config` | 编辑器皮肤配置 |
| `k4ultra_editor_theme` | 编辑器主题色 |
| `k4ultra_loading` | 启动页加载配置 |
| `kitten_theme_id` | 编辑器内置主题ID |

### 清除个性化设置

如果想恢复默认设置，可以：

1. **方法一**：在设置面板中将各项设置改回"默认"
2. **方法二**：删除配置文件目录中的 `leveldb` 文件夹
3. **方法三**：在编辑器控制台（F12）中执行：
   ```javascript
   localStorage.removeItem('k4ultra_progress_config');
   localStorage.removeItem('k4ultra_progress_skin');
   localStorage.removeItem('k4ultra_skin_config');
   localStorage.removeItem('k4ultra_editor_theme');
   location.reload();
   ```

### 发版时排除个人配置

打包发版时，个人配置不会包含在应用中，因为：
- 配置存储在用户数据目录（`%APPDATA%`），不在应用目录内
- 每个用户的配置是独立的

如果需要清除测试配置，删除上述目录即可。

## 进度条自定义

进度条是加载作品时显示的进度指示器。

### 位置选项

| 位置 | 说明 |
|------|------|
| 居中 | 默认样式，屏幕中央显示 |
| 底部 | 屏幕底部居中显示 |
| 顶部细条 | 屏幕顶部6px高的细条 |
| 底部细条 | 屏幕底部6px高的细条 |

### 风格选项

| 风格 | 说明 |
|------|------|
| 默认 | 标准进度条 |
| 霓虹 | 发光效果 |
| 渐变 | 颜色渐变动画 |
| 毛玻璃 | 半透明模糊效果 |
| 脉冲 | 呼吸灯效果 |
| 条纹 | 动态条纹 |
| 火焰 | 红黄渐变 |
| 海洋 | 蓝色渐变 |

### 形状选项

| 形状 | 说明 |
|------|------|
| 长条 | 默认长条形 |
| 圆角 | 圆角长条 |
| 胶囊 | 完全圆角 |
| 方形 | 直角方形 |
| 菱形 | 倾斜菱形 |

### 隐藏进度条

勾选"隐藏进度条"开关可以完全隐藏加载进度条。

### 自定义颜色

点击颜色选择器或预设颜色圆点可以更改进度条颜色。

### 导入进度条CSS皮肤

点击"导入进度条CSS皮肤"按钮，选择 `.css` 文件即可导入自定义进度条样式。

**进度条CSS类名参考**：
```css
/* 加载遮罩容器 */
[class*="tc-loader-wrap"] { }

/* 进度条容器 */
[class*="loader-container"] { }

/* 进度条背景 */
[class*="progress-bar-area"] { }

/* 进度条填充 */
[class*="progress-bar"] { }

/* 进度条按钮/填充色 */
[class*="progress-btns-inbox"] { }

/* 进度文字 */
[class*="progress-text"] { }

/* 加载提示 */
[class*="hint-area"] { }
```

**示例 - 自定义进度条皮肤**：
```css
/* my-progress.css */
[class*="tc-loader-wrap"] {
  background: rgba(0, 0, 0, 0.8) !important;
}

[class*="progress-btns-inbox"] {
  background: linear-gradient(90deg, #ff6b1a, #ff8c42, #ff6b1a) !important;
  background-size: 200% 100% !important;
  animation: shimmer 2s linear infinite !important;
}

@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

## 积木编辑区背景

积木编辑区（Blockly工作区）的背景可以自定义。

### 内置预设

| 预设 | 说明 |
|------|------|
| 默认 | 标准灰色背景 `#f0f0f0` |
| 纯白 | 纯白色背景 |
| 深灰 | 深色主题背景 |
| 护眼绿 | 柔和绿色背景 |
| 淡蓝 | 柔和蓝色背景 |
| 网格点 | 点状网格背景 |

### 自定义颜色

点击颜色选择器可以设置任意背景颜色。

### 导入背景图片

支持导入 PNG、JPG、GIF、WEBP 格式的图片作为背景。

**操作步骤**：
1. 进入 设置 > 个性化 > 积木编辑区背景
2. 点击"导入背景图片（PNG/JPG）"
3. 选择图片文件
4. 图片会自动拉伸填充整个工作区

### 导入CSS皮肤

点击"导入自定义积木区CSS"可以导入更复杂的样式。

**积木区CSS类名参考**：
```css
/* 工作区背景（SVG rect） */
.blocklyMainBackground { fill: #xxx; }

/* 积木分类栏 */
.blocklyToolboxDiv { background: #xxx; }

/* 积木弹出面板 */
.blocklyFlyout { background: #xxx; }

/* 积木分类项 */
.blocklyTreeRow { background: #xxx; }

/* 选中的分类 */
.blocklyTreeSelected { background: #xxx; }

/* 网格线 */
.blocklyGridLine { stroke: #xxx; }
```

| 预设 | 说明 |

### 导入CSS皮肤文件

点击"导入CSS皮肤文件"可以导入完整的自定义CSS皮肤。

### 导入皮肤包（.k4skin ZIP）

皮肤包是打包好的主题文件，格式为 `.k4skin`（实际是ZIP压缩包）。

**皮肤包结构**：
```
my-skin.k4skin (ZIP)
├── skin.json          # 必须：皮肤信息
├── main.css           # 主要CSS样式
├── blockly.css        # 积木区样式（可选）
├── toolbar.css        # 工具栏样式（可选）
└── fonts/             # 字体文件夹（可选）
    └── MyFont.ttf
```

**skin.json 示例**：
```json
{
  "name": "我的自定义皮肤",
  "version": "1.0.0",
  "author": "你的名字",
  "description": "一个酷炫的自定义皮肤"
}
```

## 制作自己的主题包

### 步骤一：创建目录结构

创建一个新文件夹，包含以下文件：
```
my-theme/
├── skin.json
└── style.css
```

### 步骤二：编写 skin.json

```json
{
  "name": "我的主题",
  "version": "1.0.0",
  "author": "我",
  "description": "我的自定义主题"
}
```

### 步骤三：编写 CSS 样式

**style.css 示例 - 完整的暗黑主题**：
```css
/* ===== 背景色 ===== */
body {
  background: #1a1a2e !important;
}

/* ===== 主题色覆盖 ===== */
:root {
  --theme-color-0: 131, 179, 255;
  --theme-color-6: 241, 245, 255;
  --theme-color-7: 245, 248, 255;
  --theme-color-8: 248, 250, 255;
  --theme-color-9: 251, 252, 255;
  --theme-color-n1: 44, 100, 230;
  --theme-color-t: 27, 40, 58;
  --theme-color-t7: 56, 91, 128;
}

/* ===== 积木编辑区 ===== */
.blocklyMainBackground {
  fill: #16213e !important;
}

.blocklyToolboxDiv {
  background: #1a1a2e !important;
}

.blocklyFlyout {
  background: #1a1a2e !important;
}

.blocklyTreeRow {
  color: #e0e0ff !important;
}

/* ===== 工具栏 ===== */
[class*="toolbar"],
[class*="header"] {
  background: #1a1a2e !important;
  color: #e0e0ff !important;
}

/* ===== 对话框 ===== */
.CUI-dialog-content,
[class*="dialog-content"] {
  background: #16213e !important;
  color: #e0e0ff !important;
}

.CUI-dialog-mask {
  background: rgba(0, 0, 0, 0.7) !important;
}

/* ===== 按钮 ===== */
.btn--1id4R.positive--3el2D {
  background: linear-gradient(135deg, #4a90d9, #357abd) !important;
}

/* ===== 侧边栏和面板 ===== */
[class*="sidebar"],
[class*="panel"] {
  background: #1a1a2e !important;
  color: #e0e0ff !important;
}

/* ===== 分类菜单 ===== */
[class*="category-menu"] {
  background: #1a1a2e !important;
}

[class*="category-menu"] * {
  color: #e0e0ff !important;
}
```

### 步骤四：打包成 .k4skin

1. 选中 `skin.json` 和 `style.css`
2. 压缩为 ZIP 文件
3. 将 `.zip` 后缀改为 `.k4skin`

### 步骤五：导入皮肤包

1. 打开 K4 Ultra 编辑器
2. 进入 设置 > 个性化 > 编辑器皮肤包
3. 点击"导入皮肤包（.k4skin ZIP）"
4. 选择你的 `.k4skin` 文件
5. 皮肤立即生效

## 常见问题

### Q: 修改后没有生效？
A: 尝试刷新编辑器（F5 或 Ctrl+R），某些修改需要重启编辑器。

### Q: 如何恢复默认设置？
A: 在设置面板中将各项改回"默认"，或删除 localStorage 配置。

### Q: 导入的CSS不生效？
A: 检查CSS语法是否正确，确保使用了 `!important` 来覆盖默认样式。

### Q: 舞台尺寸修改后没有变化？
A: 需要在编辑器中切换一下舞台比例（如从横版切到竖版再切回来）。

### Q: 如何分享我的主题？
A: 将你的 `.k4skin` 文件分享给其他人，他们可以通过"导入皮肤包"功能使用。

### Q: 配置文件在哪里？
A: Windows: `%APPDATA%\k4-ultra\Local Storage\leveldb\`
   macOS: `~/Library/Application Support/k4-ultra/Local Storage/leveldb/`
   Linux: `~/.config/k4-ultra/Local Storage/leveldb/`

### Q: 发版时如何排除个人配置？
A: 个人配置存储在用户数据目录，不在应用目录内，打包时不会包含。
